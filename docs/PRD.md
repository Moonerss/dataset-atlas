# Dataset Atlas 产品需求文档

## 1. 产品概述

Dataset Atlas 是一个用于管理科研数据集信息的网页数据库。产品面向生物信息学研究场景，帮助用户集中记录、检索、筛选和维护从 GEO、SRA、ArrayExpress、TCGA、Zenodo、Figshare 等来源搜集的数据集信息。

本阶段产品采用“个人使用优先，预留课题组协作扩展”的设计原则。前端部署在 Netlify，数据库使用 Supabase PostgreSQL。

## 2. 目标用户

### 2.1 初始用户

- 生物信息学研究者
- 需要长期搜集、比较和复用公开数据集的个人用户
- 需要追踪数据集下载、清洗和分析状态的科研人员

### 2.2 后续扩展用户

- 课题组成员
- 项目协作者
- 数据管理员

## 3. 核心目标

- 将分散的数据集信息统一存储在结构化数据库中
- 支持按疾病、物种、组织、组学类型、来源、状态等维度快速检索
- 记录数据集从发现、评估、下载、清洗到分析完成的生命周期
- 支持导入、导出和后续自动化元数据抓取
- 为后续多人协作、权限管理、操作日志保留数据库结构基础

## 4. 产品范围

### 4.1 MVP 范围

- 数据集新增、查看、编辑、归档
- 数据集列表搜索、筛选、排序
- 数据集详情页
- 标签管理
- 项目归属管理
- CSV 导入与导出
- 基础统计看板
- Supabase 数据库存储
- Netlify 前端部署

### 4.2 非 MVP 范围

- 自动下载原始测序数据
- 本地 pipeline 调度
- 复杂权限系统
- 多租户组织管理
- 自动从 GEO/SRA 抓取全量元数据
- 文件对象存储

这些能力应在 MVP 稳定后逐步加入。

## 5. 信息架构

### 5.1 页面结构

- `/`：仪表盘首页
- `/datasets`：数据集列表
- `/datasets/new`：新增数据集
- `/datasets/:id`：数据集详情
- `/datasets/:id/edit`：编辑数据集
- `/projects`：项目管理
- `/tags`：标签管理
- `/import`：CSV 导入
- `/export`：数据导出
- `/settings`：系统设置

### 5.2 导航结构

主导航建议包含：

- Dashboard
- Datasets
- Projects
- Tags
- Import / Export
- Settings

## 6. 数据集字段需求

### 6.1 基础信息

- 标题：数据集名称或论文标题
- 登录号：如 GSE、SRP、ERP、PRJNA、TCGA 项目标识等
- 来源平台：GEO、SRA、ArrayExpress、TCGA、Zenodo、Figshare、Other
- 原始链接：数据集来源 URL
- 简要描述：数据集内容摘要
- 关联论文：PMID、DOI 或论文链接

### 6.2 生物医学信息

- 物种：human、mouse、rat、other
- 疾病或表型：如 cancer、IBD、COVID-19
- 组织或细胞类型：如 blood、liver、T cell
- 组学类型：RNA-seq、scRNA-seq、ATAC-seq、WGS、WES、proteomics、metabolomics 等
- 样本数：总样本数量
- 分组信息：case/control、tumor/normal、treatment/control 等

### 6.3 数据可用性

- 数据格式：FASTQ、BAM、count matrix、h5ad、metadata、VCF 等
- 下载状态：未下载、部分下载、已下载、无法获取
- 本地路径：可选，用于记录本地数据目录
- 许可证：可选
- 数据大小：可选

### 6.4 项目管理信息

- 所属项目
- 标签
- 优先级：高、中、低
- 生命周期状态：待评估、可用、分析中、已完成、弃用、归档
- 备注
- 创建时间
- 更新时间

## 7. 功能需求

### 7.1 数据集管理

用户可以手动新增数据集，填写必要元数据。最小必填字段为标题、来源平台，以及登录号或 URL 中的至少一个。

用户可以编辑已有数据集信息。编辑时应保留创建时间，并更新修改时间。删除操作默认使用归档或软删除，不进行物理删除。

### 7.2 搜索与筛选

数据集列表应支持：

- 标题、描述、登录号、备注的关键词搜索
- 按来源平台筛选
- 按物种筛选
- 按疾病筛选
- 按组织筛选
- 按组学类型筛选
- 按下载状态筛选
- 按生命周期状态筛选
- 按项目和标签筛选
- 按更新时间、创建时间、样本数、优先级排序

### 7.3 详情页

详情页展示数据集完整信息，包括基础元数据、生物医学信息、数据可用性、项目管理信息、标签、关联论文和状态变化记录。

### 7.4 导入导出

MVP 支持 CSV 导入和 CSV/JSON 导出。导入时需要预览字段映射和错误行，避免错误数据直接进入主表。

### 7.5 统计看板

仪表盘展示：

- 数据集总数
- 按组学类型统计
- 按来源平台统计
- 按下载状态统计
- 按生命周期状态统计
- 最近新增或更新的数据集

### 7.6 协作扩展预留

虽然 MVP 以个人使用为主，但数据库应保留以下字段：

- `owner_id`
- `created_by`
- `updated_by`
- `created_at`
- `updated_at`
- `archived_at`

后续启用 Supabase Auth 后，可基于这些字段实现用户级权限和协作记录。
